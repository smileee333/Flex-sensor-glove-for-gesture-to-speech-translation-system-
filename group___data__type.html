Detailed Description

Macros for configuration of data-type to be used in generic class template used throughout the program.

max_sensor_cnt_data_type is the generic type used in class template of all the objects defined in this program. Generic data type is used to specify the data-type of value stored in gesture id or space id variable. Thus maximum supported value by the data-type specifies maxiumum possible value of gesture id or space id. Since gesture id and space id are decided as per total number of sensors and value returned by each sensor. So max_sensor_cnt_data_type is inhertly limited by the hardware design parameters including number of sensors to be used and value returned by them. For example: with hardware design including n number of sensors with each sensor returning binary value, max_sensor_cnt_data_type should be choosen in such a way that it's maximum allowed value should not exceed 2^n. If it exceeds then it would wastage of memory, which is strictly discouraged in such a memory constrained environment. However, it can be choosen with value less then that as per design requirements in which all sensors inputs are not considered.

Note
This macro usage makes the firmware scalable for any number of sensors along with any number of values returned by each sensor.
