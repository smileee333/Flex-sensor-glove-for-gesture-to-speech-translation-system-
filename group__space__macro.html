Detailed Description

Macros for configuration of gesture space meta information including, number of gesture spaces, default gesture space, space id of 1st speech space, space id of space switch gesture space.

Note
Here count value is specified instead of number of bits

◆ null_gesture_id

#define null_gesture_id   (1<<max_gesture_cnt)

Here it represents maximum value of variable storing gesture id.

It is a reserved value of gesture id, used to represent -> no gesture found case. This value represents that gesture code for which gesture id is required to be retrieved does not exist in corresponding gesture space.

◆ space_cnt_value

#define space_cnt_value   4

Here it represents total number of gesture spaces being used .

It's maximum possible value can be the maximum possible value of variable storing space id with max_space_cnt number of bits. However that meight not be case. In this project, Total 4 gesture spaces were required so max_space_cnt is kept to be 2. Following it, it has to be maximum possible value of space id. But consider a case in which 5 gesture spaces are required, in which max_space_cnt is kept to be 3 which implies maximum possible spaces to be 8. But since we are using only 5 spaces, it has to be 5 only not 8. Since we are using 1 master gesture space + 1 control gesture space + 1 speech gesture space and 1 space switch space. So space count value is kept 4 here

Since space id starts from 0, so it has to be space_cnt_value-1. following space_switch_idx is kept in all cases
