46/**
47 * @brief Continuous loop execution
48 * @details After pin mode initialization, control flow execusion take place as follows:
49 * * a. At first sampling status will be false, so CPU will go to deep sleep until, start button wakes up the CPU
50 * * b. Once CPU wakes up at start button ISR, it sets the power mode to no sleep and correspondingly disable the
51 * start button interrupt as well, until next time CPU goes to deep sleep. It also sets up the Timer.
52 * * c. as control flow leaves the ISR, it goes to Idle sleep mode.
53 * * d. The time instance at which Timer interrupt occurs, CPU goes to TIMER ISR inside which it goes to no sleep mode and
54 * then takes the sample and invoke the handler corresponding to it. Once handler is invoked, control flow leaves ISR and goes to main loop
55 * * e. Inside main loop CPU again goes to idle sleep state.
56 * * f. This process shall continue until all the sampling of words is done.
57 * * g. Once sentence translation sample is captured, Timer is stopped and correspondingly, CPU goes to deep sleep back and forth, until all the
58 * words in sentemce are not played. once all words are played. control flow leaves and again comes back to loop().
59 * * h. In loop it again goes to deep sleep as in a.
60 * * i. This process shall continue.
61 */
