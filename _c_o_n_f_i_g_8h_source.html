28/**
29 * @defgroup Data_type Data-type Macro
30 * @brief Macros for configuration of data-type to be used in generic class template used throughout the program
31 * @details
32 * max_sensor_cnt_data_type is the generic type used in class template of all the objects defined in this program. Generic data type is used to specify the data-type of value
33 * stored in gesture id or space id variable. Thus maximum supported value by the data-type specifies maxiumum possible value of gesture id or space id. Since gesture id and
34 * space id are decided as per total number of sensors and value returned by each sensor. So max_sensor_cnt_data_type is inhertly limited by the hardware design parameters
35 * including number of sensors to be used and value returned by them.
36 * For example: with hardware design including n number of sensors with each sensor returning binary value, max_sensor_cnt_data_type should be choosen in such a way that it's
37 * maximum allowed value should not exceed 2^n. If it exceeds then it would wastage of memory, which is strictly discouraged in such a memory constrained environment. However,
38 * it can be choosen with value less then that as per design requirements in which all sensors inputs are not considered.
39 * @note This macro usage makes the firmware scalable for any number of sensors along with any number of values returned by each sensor.
40 * @{
41 */
42 /// Here uint8_t is choosen as per requirment
43 #define max_sensor_cnt_data_type_macro uint8_t
44

359 * @brief Macros for configuration of Timer control registers, oncompare register, Timer counting register, Timer interrupt mode register, Timer ISR
360 * @details
361 * To promote the idea of scalable architecture, Timer specific macro are defined here, to make following parameters to be configurable
362 * * a. Choice of Timer under consideration > It gives the tuning parameter for maximum possible Timer delay as per Timer under consideration.
363 * It also gives the tuning parameter for type of timer interruption to wake-up.
364 * * b. Prescalar value > Again it gives the tuning parameter under Timer under consideration, to scale the maximum possible Timer delay.
365 * * C. On compare regiser and value > it gives the another tuning paramerer under prescale value, to specifically choose a particular Timer delay within
366 * the range of (0 - max), such that max is specified by Timer under consideration and prescale value.
367 * @{
368 */
